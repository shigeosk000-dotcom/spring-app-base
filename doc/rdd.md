# 要件定義書: ToDoリストアプリ

---

## 1. プロジェクト概要

### 1.1 背景・目的
職員が多忙な状態になってきており、作業の実施漏れによる他職員への影響も生じてきているため、個人単位で各自のToDoリストを管理し、実施漏れが生じない手助けとなるツールの作成が急務となっている

### 1.2 対象ユーザー
- 管理者: ユーザー登録と権限設定を行う
- 利用者: ToDoリストツールを利用する

---

## 2. 機能要件

### 2.1 機能一覧

| ID | 機能名 | 説明 | 優先度 |
|----|--------|------|--------|
| FR-001 | タスク追加 | タスクと関連情報を登録 | 高 |
| FR-002 | タスク削除 | 追加済みタスクを削除 | 中 |
| FR-003 | タスク編集 | タスクの登録情報を変更 | 低 |
| FR-004 | タスク完了 | 完了したタスクの消込 | 高 |
| FR-101 | ユーザー登録 | ツールの管理者・利用者ユーザーを登録 | 高 |
| FR-102 | ユーザー削除 | ツールの管理者・利用者ユーザーを削除 | 中 |
| FR-103 | ユーザー編集 | ツールの管理者・利用者ユーザーを編集 | 低 |
| FR-201 | ログイン | ログイン認証を行い、管理者／利用者の別を判定して適切な画面に遷移 | 高 |

### 2.2 ユーザーストーリー

#### US-001: ユーザー管理
- **誰が**: 管理者として
- **何を**: ユーザー管理をしたい
- **なぜ**: ToDoリスト利用者メンテナンスのため

#### US-002: ToDoリスト利用
- **誰が**: 利用者として
- **何を**: ToDoリストを利用したい
- **なぜ**: 自分のToDo管理のため

---

## 3. 非機能要件

### 3.1 パフォーマンス
- 画面表示: 2秒以内
- API応答時間: 1秒以内

### 3.2 セキュリティ
- 認証方式: セッション認証
- パスワードポリシー: 最低4文字、英数字混在
- データ暗号化: なし
- CSRF対策: 無効

### 3.2 可用性
- 稼働率： 設定しない

### 3.4 互換性
- 対応ブラウザ: Chrome
- 対応デバイス: PC / タブレット / スマートフォン
- レスポンシブ対応: 必須
- 文字コード： UTF-8 BOMなし
- 対応言語： 日本語

## 5. 制約事項
- 技術的制約: Java 17以上を使用
- 運用上の制約: 初期データは管理者1件を自動投入

## 6. 受け入れ基準

### 6.1 共通基準
- [ ] タスクの追加ができること
- [ ] タスク一覧が表示されること
- [ ] タスクの編集ができること
- [ ] タスクの削除ができること（確認ダイアログあり）
- [ ] タスクの完了/未完了の切り替えができること
- [ ] ユーザーの追加ができること
- [ ] ユーザー一覧が表示されること
- [ ] ユーザーの編集ができること
- [ ] ユーザーの削除ができること（確認ダイアログあり）
- [ ] 全画面がエラーなく表示される
- [ ] フォームのバリデーションが正しく動作する
- [ ] データの登録・更新・削除が正しく動作する
- [ ] 日本語が文字化けなく表示される
- [ ] 適切なログインID／パスワードが設定された場合のみ、利用可能であること
